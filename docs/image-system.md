# Instagram Image System

The image system is designed to produce consistent, premium Instagram/TikTok cover images without allowing the image model to write any visible text.

## Core Architecture

The system has two separate stages:

```text
OpenAI Images
  -> output/background.png
  -> Pillow composition
  -> output/instagram_release.png
```

OpenAI Images creates only a visual background. It is explicitly instructed to avoid text, letters, numbers, logos, icons, labels, screenshots, brand marks, and readable words.

Pillow then composes the final image locally. All visible text, diagrams, arrows, avatar, logos, layout, and brand elements are deterministic Python output.

## Why Text Was Removed From Image Generation

Image models can produce attractive visuals, but they are unreliable for exact typography. Earlier image generations could misspell:

- `Rodrigo Hered IA`
- provider names such as `OpenAI` or `Anthropic`
- short diagram labels
- product names

They could also introduce placeholders or random words. The current architecture avoids that by never sending final visible text to OpenAI Images. The image model only creates a text-free background, and Pillow writes every character.

## Output Files

```text
output/background.png
output/instagram_release.png
```

- `output/background.png`: text-free dark premium background generated by OpenAI Images, or by Pillow fallback if image generation fails.
- `output/instagram_release.png`: final 1080x1080 image sent to Telegram with `sendPhoto`.

## Fixed Layout

Final image size:

```text
1080x1080
```

Layout:

- Top area: strong title, maximum two lines.
- Center area: glass-effect diagram container.
- Bottom area: personal brand footer.

The composition uses a 72px margin, high-contrast typography, dark overlays for legibility, and a fixed visual hierarchy suitable for social covers.

## Templates

The system chooses one of three diagram templates.

### FLOW

Used when the release is best explained as a process.

Visual structure:

```text
Node 1 -> Node 2 -> Node 3
```

Example:

```text
OpenAI -> AWS -> Apps
```

### BEFORE_AFTER

Used for changes, fixes, deprecations, availability shifts, pricing, or latency improvements.

Visual structure:

```text
ANTES | DESPUÉS
before_text -> after_text
```

`ANTES` uses gray text. `DESPUÉS` uses white text with a subtle cyan glow to make the improvement visually dominant.

### ARCHITECTURE

Used when the release is best explained as a system view.

Visual structure:

```text
Top
  ↓
Middle
  ↓
Bottom
```

Example:

```text
OpenAI
AWS
Apps / Empresas
```

## Glass Effect

The central diagram container uses a premium glass style:

- translucent dark fill
- cyan outline
- subtle glow
- rounded corners
- clean spacing
- dark overlay for readability

Current visual values include:

```text
fill=(10, 20, 30, 180)
outline=(0, 180, 255, 120)
```

Cards and key nodes use the same visual language so each generated cover feels like part of the same brand system.

## Visual Rules

The background prompt forbids:

- text
- letters
- words
- numbers
- logos
- icons
- brand marks
- UI labels
- screenshots
- robots
- brains
- generic glowing AI art
- stock images
- fantasy visuals
- clutter

The Pillow layer enforces:

- exact title text
- exact diagram labels
- exact brand text
- local-only logos
- optional local avatar
- maximum 3-5 diagram elements
- mobile-readable text

## Personal Branding

Brand footer:

```text
Rodrigo Hered IA
AI Builder / CIO
```

Optional avatar asset:

```text
assets/brand/rodrigo.png
```

If the avatar exists, Pillow crops it to a centered square, applies a circular mask, adds a subtle cyan border/glow, and places it in the footer. If it does not exist, the footer falls back to text only.

## Local Logos

Optional logos:

```text
assets/logos/openai.png
assets/logos/anthropic.png
assets/logos/google.png
assets/logos/gemini.png
assets/logos/aws.png
assets/logos/claude.png
```

Logos are never generated by AI. They are only loaded from local files and pasted with Pillow. Missing logos do not fail the workflow.

## Anti-Error System

The image layer uses controlled helper functions before text reaches Pillow.

### `safe_image_text`

Cleans text for image use:

- removes HTML
- removes newlines
- removes placeholders like `[X]`, `[Y]`, `TODO`, `TBD`
- removes strange characters
- removes version-number noise
- limits length without aggressive word cuts
- returns a fallback if empty

### `canonical_provider_name`

Normalizes provider names:

- `Anthropic`
- `OpenAI`
- `Google`
- `AWS`

Gemini is treated as a product, not a provider.

### `build_diagram_texts`

Builds short, safe labels for each template:

- `FLOW`: controlled nodes such as provider, platform/model, apps.
- `BEFORE_AFTER`: controlled before/after pairs such as `Endpoints antiguos` -> `Integraciones actualizadas`.
- `ARCHITECTURE`: provider -> platform/product -> `Apps / Empresas`.

### `fit_wrapped_title`

Renders the title safely:

- maximum two lines
- automatically reduces font size
- avoids awkward final words like `de`, `y`, `con`, or `para`
- falls back to a shorter title if needed

## Failure Handling

If OpenAI Images fails, `create_fallback_background()` creates a dark gradient/grid background with Pillow. The final image is still generated and saved as:

```text
output/instagram_release.png
```
